1. Introduction  
AssignHub is a centralized web -based Lab Assignment Submission and Evaluation  System 
designed to streamline and automate the management of academic lab assignments in educational 
institutions. The system addresses challenges associated with manual and semi -digital assign ment 
handling, such as missed deadlines, misplaced submissions, lack of transparency, and delayed 
evaluation. AssignHub enables students to submit assignments online, faculty to evaluate and 
provide feedback efficiently, and administrators to manage academ ic structures and users through 
a single integrated platform.  
This section provides an overview of the AssignHub project and outlines the purpose, scope, and 
structure of this Software Requirements Specification (SRS). It also introduces the intended 
audie nce, key terminology, document conventions, and references required to understand the 
system requirements.  
 
1.1 Document Purpose  
The purpose of this Software Requirements Specification (SRS) document is to define the 
requirements for AssignHub ŌĆō Centralized La b Assignment Submission and Evaluation 
System (Version 1.0) . This document serves as a formal specification of the systemŌĆÖs functional 
and non -functional requirements and establishes a common understanding among stakeholders 
regarding what the system is ex pected to accomplish.  
This SRS covers the complete AssignHub system, including user authentication, assignment 
creation and submission, evaluation and feedback management, academic structure administration, 
notifications, and reporting. It is intended to a ct as a reference for system design, development, 
testing, and evaluation, without specifying implementation details.  
 
1.2 Product Scope  
AssignHub  is a web -based academic management system intended for use in educational 
institutions such as universities and colleges. The system enables faculty members to create and 
manage lab assignments, students to submit assignments electronically and track thei r status, and 
administrators to manage users, courses, and academic data.  
The primary goals of AssignHub  are to eliminate manual assignment handling, improve 
transparency in submission and evaluation, reduce administrative overhead, and ensure timely 
feedb ack to students. By providing centralized storage, role -based access, and real -time status 
updates, the system enhances academic efficiency and supports structured decision -making for 
faculty and administrators.

2. Overall Description  
2.1 Product Overview  
AssignHub  is a new, self -contained web -based Lab Assignment Submission and Evaluation System 
designed to replace traditional manual and semi -digital methods of managing academic lab work in 
educational institutions. The system provides a centralized platform for as signment creation, 
submission, evaluation, and academic data management, thereby improving efficiency, 
transparency, and reliability in academic workflows.  
AssignHub  operates within an institutional academic environment and interacts with students, 
faculty  members, and administrators. Students use the system to view assignments, submit lab work, 
and track evaluation status. Faculty members access the system to create assignments, monitor 
submissions, evaluate work, and provide feedback. Administrators manag e users, courses, batches, 
and system configurations. All assignment, submission, evaluation, and user data are maintained in 
a centralized database to ensure consistency and secure access.  
 
2.2 Product Functionality  
At a high level, the AssignHub system provi des the following major functions:  
’éĘ Secure role -based user authentication for students, faculty, and administrators  
’éĘ Creation, publication, and management of lab assignments by faculty  
’éĘ Online assignment submission and revision by students  
’éĘ Automatic recording  of submission timestamps and status  
’éĘ Evaluation of assignments with marks and feedback  
’éĘ Download of evaluated or corrected assignment files  
’éĘ Tracking of assignment submission and evaluation status  
’éĘ Notifications for assignment creation, deadlines, and evaluat ion results  
’éĘ Academic structure management including courses, batches, and faculty assignments  
’éĘ Administrative dashboards and basic reporting features

4. Other Non -functional Requirement  
 
4.1 Performance Requirement s 
The AssignHub system shall be designed to provide acceptable performance under normal 
academic usage conditions.  
’éĘ The system shall support multiple concurrent users, including students submitting 
assignments and faculty evaluating submissions simultaneously.  
’éĘ The average response time for loading pa ges such as dashboards, assignment lists, and 
submission status shall not exceed 3 seconds  under normal load.  
’éĘ Assignment file uploads and submission confirmations shall be completed within 5 seconds  
for standard file sizes.  
’éĘ Notifications related to assignm ents, deadlines, and evaluations shall be generated and 
delivered in near real time after triggering events.  
These performance requirements ensure smooth system operation during peak academic periods 
such as assignment deadlines.  
 
4.2 Safety and Security Requi rements  
The AssignHub system shall ensure the security and protection of academic data and user 
information.  
’éĘ All users shall be authenticated using secure login mechanisms.  
’éĘ User passwords shall be stored in encrypted form in the database.  
’éĘ Role-based access  control shall be enforced to prevent unauthorized access to academic or 
administrative functionalities.  
’éĘ Users shall only be able to access data relevant to their assigned roles.  
’éĘ Secure communication protocols (such as HTTPS) shall be used for data transmi ssion.  
’éĘ Uploaded assignment files shall be validated to prevent malicious or unsupported content.  
’éĘ User sessions shall be terminated automatically after a period of inactivity.  
These requirements reduce the risk of data breaches, unauthorized access, and sys tem misuse.
